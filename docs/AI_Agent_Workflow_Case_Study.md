# AI Agent Workflow Case Study

## Project Summary
Cinematic Comic Video Converter is an AI-assisted creative tool built with React, TypeScript, Vite, and OpenAI API integration.

The project explores a human-in-the-loop AI agent workflow: AI analyzes a comic page, proposes structured camera/sound/director suggestions, and the user reviews, edits, accepts, or rejects the suggestions before they become project data.

## Why this is an AI-agent workflow project
It includes:
- task decomposition into tickets
- AI-assisted implementation through Codex
- agent instructions and handoff documents
- prompt/rulebook planning
- model/provider evaluation
- human review and acceptance workflow
- manual verification and regression notes

## My workflow with Codex
I used Codex as an implementation partner:
1. Define the product problem
2. Break it into scoped tickets
3. Write clear Codex prompts
4. Review implementation reports
5. Run manual verification
6. Identify issues and create follow-up tickets
7. Update docs, roadmap, and current repo state

## Examples of agent-style work
- Designed AI page understanding contract
- Planned AI director suggestion contract
- Built review/accept/reject flow for AI suggestions
- Iterated camera-motion rulebook
- Tested different OpenAI models and provider routes
- Tuned UI after AI outputs became too cluttered
- Used documentation to preserve project memory across iterations

## What I learned
- AI agents need clear task boundaries
- Good outputs require structured context, not just prompts
- Human review is necessary when AI suggestions affect product state
- Complex AI workflows need ticketing, verification, and rulebooks
- The most important skill is turning vague goals into repeatable agent workflows
